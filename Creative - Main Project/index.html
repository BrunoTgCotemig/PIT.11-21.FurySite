Como foram criados?

Temos que voltar aos registros de bibliotecas, negócios em geral, registros policiais, fichas de pacientes e todas as informações armazenadas de maneira impressa para consultas posteriores. Foi lá que tudo começou.

Na década de 60, Os computadores se tornam parte efetiva do custo das empresas juntamente com o crescimento da capacidade de armazenamento. Cada entidade (clientes, funcionários, produtos, etc) era um arquivo de dados que eram acompanhados de um "software simples" para manipular os dados do arquivo, esses softwares permitiam realizer operações de cadastro, alteração, exclusão e consulta nos arquivos digitais.

Então, empresa IBM em 1960, investiu fortemente em pesquisas para solucionar estes problemas dos bancos de dados digitais primitivos. Vários modelos de bancos de dados surgiram nesta época, dentre eles os modelos hierárquico e rede.

Modelo Conceitual



Esta é a primeira fase da modelagem, o modelo conceitual. Nele, representaremos o mundo real por meio de uma visão simplificada dos dados e seus relacionamentos. Assim, poderemos determinar quais informações serão armazenadas no Banco de Dados. Ele é independente do SGBD(Sistema de Gerenciamento do Banco de Dados), podendo ser aplicado em qualquer software de gerenciamento de dados.

Neste nível, detalhes da implementação não aparecem; entretanto, é suficientemente detalhado ao ponto de ser possível descrever os tipos de dados requeridos, seus relacionamentos e regras de consistência.

Modelo Logico



Um modelo lógico possui conceitos que os usuários são capazes de entender, ao mesmo tempo em que não está distante do modelo físico do banco de dados.
Neste nível o projeto é independente de SGBD.
Consiste na especificação lógica dos dados em um formato adequado ao SGBD escolhido. Os tipos de dados são completamente definidos.

Modelo Físico

O modelo físico é criado a partir do modelo lógico. Nele, serão detalhadas a estrutura das informações que serão guardadas e gerenciadas pelo banco de dados. De forma generalizada, esses dados podem ser: tabelas, campos, tipos de valores ou restrições.

O intuito do modelo físico é organizar os dados de forma extremamente similar a um banco de dados real, visando facilitar sua futura implementação.

Vale a pena ressaltar que neste estágio, por se tornar um banco de dados, é necessário decidir o sistema de gerenciamento de dados e sua consequente linguagem de consulta para que o processo de modelagem prossiga conforme o resultado esperado.
